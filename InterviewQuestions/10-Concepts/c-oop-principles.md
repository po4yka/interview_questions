# Summary (EN)

OOP principles are the core ideas that define how object-oriented programs are structured: encapsulation, abstraction, inheritance, and polymorphism. They help organize code around objects and their responsibilities, improving modularity, readability, and reuse. Understanding these principles is essential for designing maintainable systems in languages like Java, Kotlin, C#, C++, Swift, and many others.

## Key Points (EN)

- Encapsulation: Hides internal state and implementation details behind a clear interface, exposing only what clients need and protecting invariants.
- Abstraction: Focuses on essential behavior (what an object does) rather than implementation details (how it does it), often via interfaces or abstract classes.
- Inheritance: Enables creating new classes based on existing ones to reuse code and model "is-a" relationships; must be used carefully to avoid tight coupling.
- Polymorphism: Allows different types to be treated uniformly via a common interface or base class, with behavior selected at runtime or compile time.
- Design impact: Correct use of these principles leads to cleaner architecture and easier testing; misuse (e.g., deep inheritance hierarchies, leaking internals) increases complexity.

## Ключевые Моменты (RU)

- Инкапсуляция: Скрывает внутреннее состояние и детали реализации за чётким интерфейсом, открывая только необходимое и защищая инварианты.
- Абстракция: Фокусируется на том, что объект делает, а не как он это делает; часто реализуется через интерфейсы или абстрактные классы.
- Наследование: Позволяет создавать новые классы на основе существующих для повторного использования кода и моделирования отношений «является»; требует аккуратного применения, чтобы избежать сильной связности.
- Полиморфизм: Даёт возможность работать с разными типами единообразно через общий интерфейс или базовый класс, выбирая поведение во время компиляции или выполнения.
- Влияние на дизайн: Корректное применение принципов ООП упрощает архитектуру и тестирование; неправильное (например, глубокие иерархии, утечка деталей реализации) усложняет систему.
