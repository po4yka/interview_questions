# Summary (EN)

Programming languages are formal languages designed to describe computations so they can be executed by machines. They define syntax (how code is written) and semantics (what code means), enabling developers to express algorithms, manipulate data, and interact with hardware and external systems. Understanding different language paradigms and ecosystems helps engineers choose appropriate tools, write clearer code, and reason about performance, safety, and maintainability.

## Key Points (EN)

- Classification: Programming languages differ by abstraction level and domain (low-level vs high-level, general-purpose vs domain-specific, compiled vs interpreted, static vs dynamic typing).
- Paradigms: Key paradigms include imperative, object-oriented, functional, procedural, and declarative; many modern languages are multi-paradigm (e.g., Kotlin, Python, JavaScript).
- Type systems: Strong vs weak, static vs dynamic typing influence safety, expressiveness, tooling, and runtime behavior.
- Tooling and ecosystem: Compilers, interpreters, build tools, package managers, and standard libraries shape productivity and typical use cases.
- Selection criteria: Choice of language often depends on platform (JVM, .NET, native, Web), performance needs, team expertise, ecosystem maturity, and interoperability requirements.

## Ключевые Моменты (RU)

- Классификация: Языки различаются по уровню абстракции и сфере применения (низкоуровневые и высокоуровневые, общего назначения и специализированные, компилируемые и интерпретируемые, со статической или динамической типизацией).
- Парадигмы: Основные парадигмы — императивная, объектно-ориентированная, функциональная, процедурная и декларативная; многие современные языки мультипарадигменные (например, Kotlin, Python, JavaScript).
- Системы типов: Сильная/слабая и статическая/динамическая типизация влияют на безопасность, выразительность, качество инструментов и поведение на этапе выполнения.
- Инструменты и экосистема: Компиляторы, интерпретаторы, системы сборки, менеджеры пакетов и стандартные библиотеки определяют продуктивность и типичные сценарии использования.
- Критерии выбора: Язык выбирают с учётом целевой платформы (JVM, .NET, native, Web), требований к производительности, опыта команды, зрелости экосистемы и требований по совместимости.
