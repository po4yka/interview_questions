# Summary (EN)

Object-Oriented Programming (OOP) is a programming paradigm based on representing software as a set of interacting objects that encapsulate data and behavior. It helps structure complex systems into reusable, modular components, improving readability, maintainability, and extensibility. OOP is widely used in languages like Java, Kotlin, C#, C++, Python, and Swift for building everything from backend services to mobile and desktop applications.

# Краткое Описание (RU)

Объектно-ориентированное программирование (ООП) — это парадигма программирования, в которой программа представляется набором взаимодействующих объектов, объединяющих данные и методы работы с ними. Такой подход помогает разбивать сложные системы на переиспользуемые и модульные компоненты, повышая читаемость, поддерживаемость и расширяемость кода. ООП широко используется в языках Java, Kotlin, C#, C++, Python, Swift и других для разработки серверных, мобильных и настольных приложений.

## Key Points (EN)

- Encapsulation: Bundles data and behavior inside objects, exposing only necessary interfaces and hiding internal implementation details.
- Inheritance: Allows creating new classes based on existing ones to reuse and extend behavior in a structured way.
- Polymorphism: Enables different implementations to be accessed through a common interface, simplifying extensibility and reducing coupling.
- Abstraction: Focuses on the essential characteristics of an entity while omitting irrelevant details, helping design cleaner APIs and models.
- Design impact: OOP influences architecture (layers, services, domain models) and is closely tied to patterns like SOLID and common design patterns (e.g., Strategy, Factory).

## Ключевые Моменты (RU)

- Инкапсуляция: Объединяет данные и поведение внутри объектов, открывая только необходимые интерфейсы и скрывая внутренние детали реализации.
- Наследование: Позволяет создавать новые классы на основе существующих для переиспользования и расширения поведения в структурированном виде.
- Полиморфизм: Дает возможность вызывать разные реализации через общий интерфейс, упрощая расширяемость и снижая связанность.
- Абстракция: Фокусируется на существенных характеристиках сущности и скрывает несущественные детали, помогая строить понятные API и модели.
- Влияние на дизайн: ООП влияет на архитектуру (слои, сервисы, доменные модели) и тесно связано с принципами SOLID и шаблонами проектирования (например, Strategy, Factory).
