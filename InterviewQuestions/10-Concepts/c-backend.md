# Summary (EN)

Backend (server-side) is the part of an application responsible for processing business logic, accessing data storage, integrating with external services, and exposing APIs to clients (web, mobile, other systems). It runs on servers, handles requests from the frontend, enforces security and validation rules, and ensures reliability, scalability, and performance of the system as a whole.

# Краткое Описание (RU)

Backend (серверная часть) — это часть приложения, отвечающая за выполнение бизнес-логики, доступ к хранилищам данных, интеграцию с внешними сервисами и предоставление API клиентам (web, mobile и другим системам). Он работает на серверах, обрабатывает запросы от фронтенда, обеспечивает безопасность, валидацию данных, надежность, масштабируемость и производительность всей системы.

## Key Points (EN)

- Separation of concerns: backend focuses on data processing, business rules, security, and integration, while the frontend handles presentation and user interaction.
- Core technologies: typically implemented using languages like Java, Kotlin, Python, JavaScript/TypeScript (Node.js), Go, C#, etc., along with frameworks (Spring, Django, Express, ASP.NET, etc.).
- Data and APIs: exposes REST, GraphQL, gRPC or other APIs; interacts with databases (SQL/NoSQL), caches, message brokers, and file storage.
- Non-functional requirements: responsible for authentication/authorization, performance, scalability, observability (logging, metrics, tracing), and fault tolerance.
- Architectures: often built using layered, hexagonal, microservices, or serverless architectures depending on system complexity and scalability needs.

## Ключевые Моменты (RU)

- Разделение ответственности: backend отвечает за обработку данных, бизнес-правила, безопасность и интеграции, тогда как frontend занимается отображением и взаимодействием с пользователем.
- Основные технологии: обычно реализуется на языках Java, Kotlin, Python, JavaScript/TypeScript (Node.js), Go, C# и др., с использованием фреймворков (Spring, Django, Express, ASP.NET и т.п.).
- Данные и API: предоставляет REST, GraphQL, gRPC и другие API; работает с базами данных (SQL/NoSQL), кэшами, брокерами сообщений и файловым хранилищем.
- Нефункциональные требования: отвечает за аутентификацию/авторизацию, производительность, масштабируемость, наблюдаемость (логи, метрики, трейсинг) и устойчивость к сбоям.
- Архитектуры: часто строится на основе многоуровневой архитектуры, hexagonal-архитектуры, микросервисов или serverless-подхода в зависимости от сложности и требований к масштабированию.
