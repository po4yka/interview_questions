# Summary (EN)

OOP concepts (Object-Oriented Programming concepts) are a set of principles for structuring programs around objects that combine data and behavior. They help create modular, reusable, and maintainable code by modeling real-world entities and relationships. Core OOP concepts typically include encapsulation, inheritance, polymorphism, and abstraction, and are used in languages like Java, C#, C++, Kotlin, Python, and many others.

# Краткое Описание (RU)

Концепции ООП (объектно-ориентированного программирования) — это набор принципов, по которым программы строятся вокруг объектов, объединяющих данные и поведение. Они помогают создавать модульный, переиспользуемый и сопровождаемый код, моделируя реальные сущности и связи между ними. Ключевые концепции ООП включают инкапсуляцию, наследование, полиморфизм и абстракцию и применяются в языках Java, C#, C++, Kotlin, Python и многих других.

## Key Points (EN)

- Encapsulation: Bundling data and methods together in a class and controlling access (e.g., via access modifiers) to protect internal state and enforce invariants.
- Inheritance: Defining new classes based on existing ones to reuse behavior and express "is-a" relationships, while being mindful of overuse and tight coupling.
- Polymorphism: Treating different implementations through a common interface or base type, enabling flexible code (e.g., method overriding and interface-based design).
- Abstraction: Hiding unnecessary implementation details and exposing clear, minimal interfaces focused on what an object does rather than how it does it.
- Object composition: Combining objects to build complex behavior ("has-a" relationships), often preferred over deep inheritance hierarchies for better flexibility and testability.

## Ключевые Моменты (RU)

- Инкапсуляция: Объединение данных и методов в классе и управление доступом (через модификаторы доступа) для защиты внутреннего состояния и соблюдения инвариантов.
- Наследование: Определение новых классов на основе существующих для повторного использования поведения и выражения отношений типа "is-a", при этом важно избегать чрезмерного и жёсткого связывания.
- Полиморфизм: Возможность работать с разными реализациями через общий интерфейс или базовый тип, что повышает гибкость кода (например, переопределение методов, интерфейсно-ориентированный дизайн).
- Абстракция: Сокрытие ненужных деталей реализации и предоставление чётких, минимальных интерфейсов, сфокусированных на том, что объект делает, а не как он это делает.
- Композиция объектов: Построение сложного поведения путём объединения объектов (отношения "has-a"), часто предпочтительнее глубокой иерархии наследования для лучшей гибкости и тестируемости.
