# Summary (EN)

The **Data Access Object (DAO)** pattern is a structural pattern that isolates the application/business layer from the persistence layer (usually a relational database or an API). It defines an abstract API to perform CRUD operations on specific data objects without exposing details of the database implementation.

# Сводка (RU)

Паттерн **Data Access Object (DAO)** - это структурный паттерн, который изолирует слой приложения/бизнес-логики от слоя персистентности (обычно реляционной базы данных или API). Он определяет абстрактный API для выполнения CRUD операций над конкретными объектами данных, не раскрывая деталей реализации базы данных.
